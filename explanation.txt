

# Описание процесса тестирования роли Apache с помощью Molecule

Тестирование проводилось в среде WSL2 (Ubuntu) с использованием драйвера Docker. Ниже описана последовательность действий:

1. **Инициализация окружения**:
   Установил Molecule и плагин для работы с Docker командами `pipx install molecule` и `pipx inject molecule 'molecule-plugins[docker]'`.

2. **Настройка инфраструктуры (molecule.yml)**:
   - В секции `platforms` указал образ `geerlingguy/docker-ubuntu2204-ansible:latest` с поддержкой systemd.
   - Добавил параметры `privileged: true` и `command: /sbin/init` для корректного запуска сервиса Apache внутри контейнера.
   - Настроил проброс `cgroup`, чтобы избежать ошибок при старте служб в Docker Desktop.

3. **Подготовка сценария (converge.yml)**:
   - Настроил подключение роли через `include_tasks: "../../tasks/main.yml"`.
   - В блоке `handlers` явно прописал обработчики `Reload Apache` и `Restart Apache`, так как динамическое подключение через
     задачи не регистрировало их автоматически.

4. **Настройка верификации (verify.yml)**:
   - Добавил задачи по созданию временной директории `/var/www/localhost/wordpress` и тестового файла `index.html`, чтобы Apache было что отдавать.
   - Настроил проверку доступности через модуль `uri` по адресу `http://127.0.0.1/` с обязательным заголовком `Host: localhost`.

5. **Решение проблем с зависимостями**:
   - Столкнулся с ошибкой отсутствия коллекций `community.docker` и `ansible.posix` внутри окружения Molecule.
   - Решил проблему созданием символических ссылок из системных папок Ansible в виртуальное окружение Molecule (`pipx`).

6. **Запуск тестов**: ( molecule test из за качества интернета тут у нас в деревне сегодня что-то капризничает).
   - Выполнил команду `molecule create` для поднятия контейнера.
   - Запустил `molecule converge` для наката роли.
   - Провел финальную проверку командой `molecule verify`.
   - Весь цикл успешно завершен со статусом `Successful`.

7. **Результат**:
   Лог успешного прохождения тестов сохранен в файл `molecule_verify_report.txt`
